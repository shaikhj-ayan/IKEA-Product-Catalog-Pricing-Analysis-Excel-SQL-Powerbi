## 📖 Overview

In this project, I worked with IKEA product data to study prices, categories, and product details.

I cleaned the data using Excel and SQL, did analysis using SQL, and created a dashboard in Power BI to show insights.



## 💼 Business Problem

* Large product data makes it hard to understand pricing
* Different categories follow different pricing styles
* Hard to identify premium and budget products
* No clear view of category performance
* Need better decisions using data



## 📂 Dataset

* Source: Hugging Face
* Dataset: [IKEA US Products 2025 Dataset](https://huggingface.co/datasets/jeffreyszhou/ikea-us-products-2025?utm_source=chatgpt.com)
* The dataset contains ~30,000 IKEA US products with details like product title, price, category, description, and more ([Hugging Face][1])

### Data Preparation

* Cleaned basic formatting issues in Excel
* Fixed column names and data types in SQL
* Checked duplicate rows
* Handled blank and empty values
* Standardized category and subcategory names
* Cleaned product titles
* Fixed encoding issues (special characters)
* Corrected spelling mistakes
* Split combined category values into proper columns

Final data was clean and ready for analysis.

## 🔍 Research Questions

* What is the most expensive product?
* Which category has the most products?
